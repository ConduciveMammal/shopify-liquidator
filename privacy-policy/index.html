This policy explains what data Theme Liquidator processes, why it is used, and how merchants can request deletion or contact the developer. Theme Liquidator is operated by Liam Merlyn through Merlyn Design Works. Website: merlyndesignworks.co.uk The app processes only the data needed to authenticate a merchant, list themes, and delete themes selected by that merchant. Store identifiers such as the shop domain. OAuth and API access tokens needed to call the Theme API on behalf of the merchant. Theme metadata returned by Shopify, such as theme names, roles, processing status, and update timestamps. Basic operational metadata such as granted scopes and authentication timestamps. To authenticate the merchant and establish a session between the CLI and the hosted broker. To show the merchant which themes are eligible for deletion. To submit deletion requests only for themes the merchant explicitly confirms. To satisfy Shopify platform requirements, including mandatory compliance webhooks. Local CLI configuration is stored on the merchant's own machine. Sensitive credentials are intended to be stored in the operating system credential store where available. The hosted broker stores short-lived authentication session data and app installation tokens only as needed to complete authentication and serve authenticated CLI requests. Theme Liquidator is not designed to store customer personal data. If Shopify sends a compliance webhook, it is handled to meet platform obligations. Data is not sold. Data is shared only with service providers needed to operate the app infrastructure and with Shopify as part of normal platform operation. Merchants can revoke app access at any time from Shopify. Revoking access prevents any further authenticated API use by the app. For privacy enquiries or deletion requests, use the contact page at merlyndesignworks.co.uk/contact/.
